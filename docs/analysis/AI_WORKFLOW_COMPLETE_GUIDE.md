## 💡 核心洞察

### 1. 上下文为王

```
成功的AI编程 = 清晰的上下文 + 结构化流程 + 迭代验证

不是依赖AI"魔法"，而是给AI正确的信息：
- 精确的文件路径和内容
- 库版本和文档
- 已知问题和边缘情况
- 可执行的验证步骤
```

### 2. 分步验证胜过一次性完成

```
❌ 坏方法：
"构建整个认证系统"
→ 结果：可能大规模返工

✅ 好方法：
Task 1.1: 数据库schema
审查 → 批准 ↓
Task 1.2: API端点
审查 → 批准 ↓
Task 1.3: 前端集成
审查 → 批准 ↓

→ 结果：每步可控，容易调试
```

### 3. 工具组合使用

```
不同任务用不同工具：

快速原型 → Bolt.new
日常开发 → Cursor
复杂重构 → Claude Code
文档规划 → ChatGPT GPT

不要试图用一个工具做所有事
```
